📘 What is Base64?

Read this before starting — every level builds on this concept.

Base64 is an encoding scheme — not encryption — that converts binary or text data into a string made of only 64 printable ASCII characters. It exists because some systems (old email protocols, URLs, JSON, config files) can't safely carry raw binary data, so Base64 repackages that data into safe, printable text. It is fully reversible with no key: anyone can decode it instantly, which is exactly why CTFs love using it as a first obfuscation layer.
Base64 alphabet: ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/   (padding character: =)
🔍 How to recognize it
Only letters, digits, + and /, often ending in = or == padding.
🔄 Reversible
Decoding requires no password or key — it's just a lookup table.
📦 3 bytes → 4 chars
Every 3 raw bytes become 4 Base64 characters, padded if needed.
🧩 Stacks
Output can be Base64 again ("multi-layer") or hide inside other formats.
